Модуль Neighbor Info отправляет в Mesh сведения о 0-hop соседях каждого узла. Параметры: Enabled, Update Interval и Transmit Over LoRa.

:::info информация
Neighbor Info — только для информации и не влияет на маршрутизацию Mesh. Модуль необязателен и по умолчанию выключен.
:::

Чтобы экономить эфир, Meshtastic не отслеживает маршрут доставки пакета, кроме режима Traceroute. При включении этого модуля и опции "Transmit Over LoRa" ваш узел периодически отправляет пакет со списком прямых соседей и качеством линка (SNR). Если включить это на всех узлах, можно собрать граф всей сети и увидеть, как всё связано. На момент написания готовых визуализаторов нет, но можно собрать данные, например, через MQTT.

Для работы используйте Firmware 2.2.0 или выше. Начиная с 2.3.2, модуль определяет соседей даже если на других узлах модуль выключен.

## Параметры конфигурации модуля Neighbor Info {#neighbor-info-module-config-values}

### Enabled {#enabled}

Включает модуль Neighbor Info.

### Update Interval {#update-interval}

Как часто отправлять данные о соседях в Mesh. Минимум — 4 часа (14400 секунд). По умолчанию — 6 часов (21600 секунд).

### Transmit Over LoRa {#transmit-over-lora}

Доступно начиная с Firmware 2.5.13. По умолчанию данные о соседях отправляются только в MQTT и подключённое приложение. Если включено, отправка идёт по первичному Channel через LoRa. Доступно только если первичный Channel не является публичным каналом с ключом и именем по умолчанию.

:::info информация

Все настройки модуля Neighbor Info доступны в python CLI версии 2.2.0 и выше.

:::

Примеры команд:

```shell title="Включить/выключить модуль Neighbor Info"
meshtastic --set neighbor_info.enabled true
meshtastic --set neighbor_info.enabled false
```

```shell title="Задать интервал 5 часов"
meshtastic --set neighbor_info.update_interval 18000
```

```shell title="Включить Transmit Over LoRa"
meshtastic --set neighbor_info.transmit_over_lora true
```

```shell title="Показать конфиг модуля Neighbor Info"
meshtastic --get neighbor_info
```
