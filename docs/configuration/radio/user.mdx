Опции User Config: Long Name, Short Name, Is Licensed и Is Unmessageable. User Config использует admin message с `User` protobuf.

## Параметры User Config {#user-config-values}

### Длинное имя {#long-name}

Персонализированное имя вашего устройства.

По умолчанию генерируется автоматически.

Если у вас есть радиолюбительская лицензия и включён `IsLicensed`, здесь укажите свой позывной.

### Короткое имя {#short-name}

Короткий идентификатор вашего устройства.

По умолчанию генерируется автоматически.

### Есть лицензия (ham) {#is-licensed-ham}

Если у вас есть радиолюбительская лицензия и вы учитываете [права и ограничения](/docs/faq/#amateur-radio-ham) использования Meshtastic с ham-лицензией, включите этот флаг.

По умолчанию выключено.

При включении `IsLicensed` также проверьте связанные настройки:

- User: `LongName` (должно быть вашим позывным)
- Channel: `PSK` (должно быть пусто — шифрование отключено)

### Не принимать личные сообщения {#is-unmessageable}

Можно ли отправлять узлу личные сообщения напрямую.

По умолчанию: False, кроме ролей, которые обычно не отслеживаются для личных сообщений: ROUTER, ROUTER_LATE, REPEATER, SENSOR, LOST_AND_FOUND, TRACKER, TAK_TRACKER

#### CLI

:::note заметка

Все опции User Config доступны в Python CLI. Примеры команд ниже:

:::

См. инструкции по [Enabling Ham License](/docs/software/python/cli/usage#ham-radio-support)

:::tip совет

Так как устройство перезагружается после каждой команды из CLI, при настройке нескольких значений в одном разделе удобнее объединить команды в одну строку.

```shell title="Пример:"
meshtastic --set-owner 'your node name' --set-owner-short  'NODE'
```

:::

```shell title="Установить LongName"
meshtastic --set-owner 'your node name'
```

```shell title="Установить ShortName"
meshtastic --set-owner-short  'NODE'
```

```shell title="Включить Ham-режим, задать имя = позывной, отключить шифрование"
meshtastic  --set-ham 'CALLSIGN'
```
