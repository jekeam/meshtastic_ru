Параметры Bluetooth: Enabled, Pairing Mode и Fixed PIN. Настройка Bluetooth отправляется админ‑сообщением `Config.Bluetooth` (protobuf).

:::note заметка
ESP32: Bluetooth отключается, если включен Wi‑Fi. Приоритет у Wi‑Fi.
:::

## Параметры Bluetooth {#bluetooth-config-values}

### Включено {#enabled}

Включает Bluetooth.

### Режим привязки {#pairing-mode}

Выберите режим привязки.

`RANDOM_PIN` генерирует случайный PIN во время работы.
`FIXED_PIN` использует фиксированный PIN, который нужно задать отдельно.
`NO_PIN` отключает проверку PIN.

### Режим привязки по умолчанию {#default-pairing-mode}

Режим по умолчанию выбирается в зависимости от того, подключён ли к устройству экран при первом запуске (или со сброшенным состоянием устройства), если вручную не задан через настройки Bluetooth.

- **Экран подключён:** Если устройство загружается впервые (или со сброшенным состоянием) и обнаружен экран, режим по умолчанию будет `RANDOM_PIN`. Если экран снять после загрузки, режим `RANDOM_PIN` останется, пока вы вручную не смените его на `FIXED_PIN` или `NO_PIN`. Рекомендуется изменить режим до снятия экрана.
- **Экрана нет:** Если устройство загружается впервые (или со сброшенным состоянием) и экран не обнаружен, режим по умолчанию будет `FIXED_PIN` со значением ниже, если вручную не задать другое.

### Фиксированный PIN {#fixed-pin}

Если выбран `FIXED_PIN`, значение по умолчанию — **123456**. Для других режимов это число игнорируется. Можно задать свой 6‑значный PIN в настройках Bluetooth.

:::danger предупреждение
Настоятельно рекомендуется сменить PIN по умолчанию для `FIXED_PIN`. Дефолтный код — серьёзный риск для безопасности.
:::

:::note заметка

#### CLI

:::note заметка

Все параметры Bluetooth доступны в Python CLI. Примеры команд ниже:

:::

|      Параметр       |        Допустимые значения          | По умолчанию |
| :-----------------: | :----------------------------------: | :----------: |
| `bluetooth.enabled` |           `true`, `false`            |    `true`    |
|  `bluetooth.mode`   | `RANDOM_PIN`, `FIXED_PIN`, `NO_PIN`  | `RANDOM_PIN` |
| `bluetooth.fixed_pin` |        `integer` (6 digits)         |   `123456`   |

:::tip подсказка

Так как устройство перезагружается после каждой команды из CLI, при изменении нескольких значений удобнее объединять их в одну команду.

```shell title="Пример:"
meshtastic --set bluetooth.enabled true  --set bluetooth.fixed_pin 111111
```

:::

```shell title="Включить/выключить модуль Bluetooth"
meshtastic --set bluetooth.enabled true
meshtastic --set bluetooth.enabled false
```

```shell title="Задать фиксированный PIN"
meshtastic --set bluetooth.mode FIXED_PIN
meshtastic --set bluetooth.fixed_pin 111111
```
