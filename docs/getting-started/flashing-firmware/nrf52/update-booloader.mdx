Если есть проблемы с обновлением или прошивкой новых версий Meshtastic Firmware и на устройстве nRF52 не стоит актуальный Bootloader, обновление Bootloader может решить их.

Чтобы узнать версию Bootloader, переведите устройство в DFU-режим. Затем откройте смонтированный диск, который появится на компьютере, и посмотрите файл `INFO_UF2.TXT`.

## Обновление Bootloader

Ниже — как обновить Bootloader.

### Метод 1: через UF2-файл (рекомендуется)

Для вашего устройства нужен подходящий пакет Bootloader. Ссылки ниже:

- [Lilygo T-Echo](https://github.com/meshtastic/firmware/raw/master/bin/update-lilygo_techo_bootloader-0.6.1_nosd.uf2)
- [RAK4631](https://github.com/RAKWireless/WisBlock/raw/master/bootloader/RAK4630/Latest/wiscore_rak4631_board_bootloader-0.4.3_s140_6.1.1.uf2)
- [Seeed Tracker 1000-E](https://github.com/adafruit/Adafruit_nRF52_Bootloader/releases/download/0.9.2/update-t1000_e_bootloader-0.9.2_nosd.uf2)
- [Generic Meshtastic 6.1.1 для DIY](https://github.com/meshtastic/firmware/raw/master/bin/generic/update-Meshtastic_6.1.0_bootloader-0.9.2_nosd.uf2)
- [Generic Meshtastic 7.3.0 для DIY](https://github.com/meshtastic/firmware/raw/master/bin/generic/update-Meshtastic_7.3.0_bootloader-0.9.2_nosd.uf2)

1. Скачайте нужный UF2-файл Bootloader.
2. Подключите устройство к компьютеру по USB.
3. Включите режим Bootloader, быстро дважды нажав кнопку RESET. Для Seeed Tracker 1000-E нужно зажать пользовательскую кнопку и дважды быстро подключить USB-кабель к MagSafe-порту. На компьютере устройство появится как съёмный диск.
4. Перетащите скачанный UF2-файл на этот съёмный диск. Устройство обновит Bootloader и перезагрузится.
5. После перезагрузки обновление завершено. На устройстве установлен актуальный Bootloader, можно переходить к [прошивке Firmware](/docs/getting-started/flashing-firmware/nrf52/drag-n-drop/).

### Метод 2: через adafruit-nrfutil

:::caution

В отличие от загрузки через UF2, adafruit-nrfutil не проверяет, подходит ли пакет Bootloader для вашего устройства. Если прошить не тот Bootloader, можно получить «кирпич». Также проверьте SHA256-сумму перед прошивкой.

:::

:::info

Эти шаги предполагают, что у вас уже установлены Python и pip. Если нет — установите свежий Python (включает pip) с [Python.org](https://www.python.org/downloads/).

:::

Для вашего устройства нужен подходящий пакет Bootloader. Ссылки ниже:

- [Lilygo T-Echo](https://github.com/meshtastic/firmware/raw/master/bin/lilygo_techo_bootloader-0.6.1.zip) SHA256: 85d8a334bbf82802d712e183f29ec5215f06786ca88914687c437aceab75d9cf
- [RAK4631](https://github.com/RAKWireless/WisBlock/raw/master/bootloader/RAK4630/Latest/wiscore_rak4631_board_bootloader-0.4.3_s140_6.1.1.zip) SHA256: 4a216ad2be8be23b80371a11753677c850c5711d3b85129390a416fc47ea0910
- [Seeed Tracker 1000-E](https://github.com/adafruit/Adafruit_nRF52_Bootloader/releases/download/0.9.2/t1000_e_bootloader-0.9.2_s140_7.3.0.zip) SHA256: 8c69f0d43a7aac925055451d7262682d6926d4cfb7ea8240b466dc8f16a692ba
- [Generic Meshtastic 6.1.1 для DIY](https://github.com/meshtastic/firmware/raw/master/bin/generic/Meshtastic_6.1.0_bootloader-0.9.2_s140_6.1.1.zip) SHA256: ecebecea849ab79d09517dd4f6ff98de5647fe275b0b4d525501e6c29cb5a586
- [Generic Meshtastic 7.3.0 для DIY](https://github.com/meshtastic/firmware/raw/master/bin/generic/Meshtastic_7.3.0_bootloader-0.9.2_s140_7.3.0.zip) SHA256: 9a38edf4e974a6f705c41b296499a4fc57682ec9bb686eecd9f3d8d02fc6ffcf

1. Откройте терминал или командную строку и установите adafruit-nrfutil командой:

```bash
pip install adafruit-nrfutil
```

2. Скачайте нужный zip-пакет.
3. Подключите устройство к компьютеру по USB.
4. В терминале перейдите в папку с zip-пакетом Bootloader и выполните команду, заменив /dev/ttyACM0 на нужный порт вашего устройства (в Windows это может быть COMx):

```bash
adafruit-nrfutil --singlebank --touch 1200 --verbose dfu serial --package <downloaded file>.zip -p /dev/ttyACM0 -b 115200
```

5. После окончания процесса обновление завершено. На устройстве установлен актуальный Bootloader, можно переходить к [прошивке Firmware](/docs/getting-started/flashing-firmware/nrf52/drag-n-drop/).

### Метод 3: через Debugger

Если способы выше не подошли и железо позволяет (есть пины SWD), можно прошить Bootloader напрямую через Debugger, например DAPLink или J-Link. Пример для RAK4631 — в [инструкции по Debugger](https://meshtastic.org/docs/getting-started/flashing-firmware/nrf52/convert-rak4631r/#debugger).
