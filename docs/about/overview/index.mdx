## Как это работает {#how-it-works}

Когда вы отправляете сообщение в приложении Meshtastic, оно передаётся на радио по Bluetooth, Wi‑Fi/Ethernet или по Serial. Затем радио передаёт сообщение в эфир. Если в течение таймаута не получено подтверждение от другого устройства, сообщение будет повторно отправлено до трёх раз.

Приёмник, получив пакет, проверяет, видел ли он это сообщение раньше. Если да — игнорирует. Если нет — ретранслирует.

При каждой ретрансляции радио уменьшает «hop limit» на единицу. Если приходит пакет с нулевым hop limit, сообщение больше не ретранслируется.

Радио хранит небольшой буфер пакетов (около 30) на случай отсутствия подключения к клиентскому приложению. Если буфер заполнен, самые старые пакеты заменяются только новыми текстовыми сообщениями.

## Что такое Mesh? {#what-is-a-mesh}

На радиоуровне Mesh в Meshtastic — это набор Nodes, использующих одинаковые параметры LoRa: spreading factor, центральную частоту и полосу. Одна Node может находиться только в одном радиомеше; она не видит и не обрабатывает сообщения от Nodes с другими значениями этих параметров. Чтобы сформировать Mesh, Nodes должны использовать одинаковые значения.

Эти значения объединены в «presets», которые можно выбрать в разделе настроек LoRa. Presets упрощают настройку одинаковых радиопараметров для всех Nodes.

Сверху над этим радиомешем находятся Channels. Логический Mesh образуется Channel с определённым именем и ключом шифрования. Канал по умолчанию в радиомеше — Channel 0 с пустым «name» и ключом шифрования AQ==.

Одна Node может состоять максимум в 8 Channels внутри радиомеша. Можно создать свой Channel для конкретной группы. Только Nodes, настроенные с тем же именем Channel и ключом шифрования, смогут читать и показывать сообщения в этом Channel. Однако все Nodes в радиомеше получают и могут ретранслировать сообщения (в зависимости от их Role) независимо от настроек Channel для сообщения.
